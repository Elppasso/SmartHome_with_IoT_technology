## ✨ Основні можливості

### 📊 Дашборд
- Автоматичне оновлення даних кожні 30 секунд
- Картки середньої температури по кімнатах
- Таблиця інфраструктури (будинки → кімнати → пристрої → сенсори)
- Список останніх подій з хронологією

### 🔧 Керування пристроями
- Деревоподібна навігація (будинки → кімнати → пристрої)
- Створення/видалення будинків та кімнат через модальні вікна
- Додавання пристроїв з можливістю одразу створити сенсор (10 типів)
- Зміна статусу пристрою (Увімкнено/Вимкнено) з миттєвим оновленням

### 📈 Моніторинг сенсорів
- Фільтрація за типом сенсора (Temperature, Humidity, Motion, WaterLeak тощо)
- Інтерактивний лінійний графік (Chart.js) з періодами: 1 год, 24 год, 7 днів, 30 днів
- Статистика: поточне, середнє, мінімальне, максимальне значення
- Експорт даних у CSV та друк графіка

### ⚙️ Автоматизація
- Перегляд правил типу `IF ... THEN ...`
- Відображення умови, дії та цільового пристрою

### 📋 Журнал подій
- Автооновлення кожні 10 секунд
- Показано останні 50 подій

## 🏗 Архітектура проєкту

![Архітектура](https://via.placeholder.com/800x400?text=Діаграма+архітектури)

*(Тут можна вставити діаграму взаємодії компонентів)*

### Технологічний стек

| Компонент       | Технологія                        |
|----------------|-----------------------------------|
| **Backend**    | Python 3.8+, Flask, pyodbc        |
| **Frontend**   | HTML5, Bootstrap 5, JavaScript ES6 |
| **Візуалізація** | Chart.js                         |
| **База даних** | Azure SQL Database (MS SQL Server) |
| **Конфігурація** | python-dotenv (.env)             |

### Схема взаємодії

1. **Клієнт** (браузер) надсилає запити до Flask-сервера.
2. **Flask** маршрутизує запити та викликає методи `SmartHomeDB`.
3. **SmartHomeDB** створює нове з’єднання з Azure SQL через `pyodbc`, виконує SQL-запит, повертає результат у вигляді JSON.
4. **Клієнт** отримує JSON та динамічно оновлює DOM (без перезавантаження сторінки).
